# 1. Назначение программы.

Данная программа считает количество максимумов в массиве.

### Анализ изменения с помощью *git status* и *git diff*

![Снимок_экрана__128_](uploads/de48661e65140b67c87dd80390e3c2a7/Снимок_экрана__128_.png)
![Снимок_экрана__126_](uploads/2f0f1f1f479dac6b5be5806ac9dd4fea/Снимок_экрана__126_.png)

С помощью команды *git status* мы видим, что файл *main.py* изменен, но не в индексе для коммита.

С помощью команды *git diff* мы видим, что в данный файл мы добавили строки. Если в начале строки стоит '-'это значит, что данная строка удалена. Если стоит '+' это значит, что данная строка добавлена. Если стоит пробел это значит, что строка без изменений.

Записи *'---a/main.py'* и *'+++b/main.py'* означают, что файл main.py был изменен.

Запись *'@@ -11,12 +11,26 @@'* означает, что из файла извлечено 12 строк, начиная с 11, и добавлено 26, начиная с 11.

### Зафиксированные изменения с помощью *git log*.

![Снимок_экрана__141_](uploads/225df75387f0836160f851ae363f0506/Снимок_экрана__141_.png)

С помощью команды *git log* мы получили список коммитов в обратном хронологическом порядке.
 
Формат:
```
commit <SHA-1 коммита>
Author: <user.name> <<user.email>>
Date:   <дата создания>
```

### Зафиксированные изменения с помощью *git log --name-status*.

![Снимок_экрана__146_](uploads/048eea9f95a4583d8e64d5927c25cede/Снимок_экрана__146_.png)

С помощью команды *git log --name-status* мы получаем так же список файлов, которые были изменены/добавлены/удалены (A - добавлены, D - удалены, M - изменены).

### Зафиксированные изменения не за весь период, а между двумя определенными ревизиями с помощью команды *git log a..b* 

![Снимок_экрана__145_](uploads/97c9a503cacd43bfbbfabe96cfe6ebf0/Снимок_экрана__145_.png)

Команда *git log a..b*,где a и b это номера ревизий, позволяет отследить изменения между определенными ревизиями.

### Зафиксированное изменения одного и того же файла с помощью команды *git diff a b <имя файла>*

![Снимок_экрана__148_](uploads/2fd233c3c3681e8288a20bc11bc22013/Снимок_экрана__148_.png)

С помощью команды *git diff a b <имя файла>*, где a и b номера ревизий, можно отследить изменения конкретного файла.
